
Я сделал тебе небольшой рефакторинг. Также я оптимизировал структуру проекта

============== РЕКОМЕНДАЦИИ ==============

    1. Избегай сильно больших отступов в коде
    2. Закрывай теги так, чтобы отображались линии. Иначе не будет видно, что куда принадлежит.
    3. Придерживайся одной структуры во всем проекте. Структурируй его правильно
    4. Избегай повторения одного и того же кода несколько раз. Если он повторяется, выноси его в отдельный модуль и потом импортируй куда тебе нужно
       - Пример: Я вынес твои скрипты в отдельный файл с джаваскриптами.
       - Писать джаваскрипты в html это плохая практика
            - Почему: В отделе айтишников один может специализироваться на css, другой - на верстке html, третий - на джаваскрипте.
                      Дабы не усложнять никому работу, лучше разделять такие элементы по отдельным файлам.

    5. Хорошей практикой считается использовать стандартные имена для страниц. 
       - Пример:  hotel_home лучше назвать index (значит: "Главная страница")
            - Почему: Apache, например, использует это имя по дефолту. Кроме того, всем понятнее что страница "индекс" - Это главная. hotel_home может быть чем угодно

    6. Выноси SQL - стейтменты в отдельные .sql файлы, а затем импортируй их
       - Почему: См. строку 12 этого документа
    7. Комментарии. Твои коллеги скажут тебе огромное спасибо, так как не надо искать что-то несколько часов. Сразу все видно.
    8. Обязательно пиши README по типу этого. Причина та же что и в пункте 7.
